# Frontend (React / TypeScript / Vite) Context

## What this is

Vite 8 (Rolldown) + React + TypeScript 6 + React Router SPA. 37 components in `src/components/`, 14 islands in `src/islands/`, 9 page wrappers in `src/pages/`, 1 layout in `src/layouts/`, 8 hooks in `src/hooks/`, 1 shim in `src/shims/`, 3 stores in `src/contexts/`. 1248 Vitest 4 tests across 86 test files. `npm run dev` proxies to FastAPI; `npm run build` outputs to `frontend/dist/`. See `docs/design-react-component-library.md` for build sequence.

## Build & type-checking

- **Bundle size gate** â€” `npm run size` (after `npm run build`) measures gzip size of the **live SPA in English** and checks it stays under 220 kB. Current ~196 kB (Node 24 gzip output; ratcheted **down** from 230 kB on 1 Jul 2026 after fixing the dual-import that defeated island code-splitting, then a further drop on 3 Jul 2026 when the `preflight-*` locale chunk was finally added to the exclude glob â€” see below). CI enforces this. The glob explicitly excludes lazy locale chunks â€” it must name **all 9 namespaces** (`common/settings/desktop/enums/cli/doctor/pipeline/preflight/server`); `preflight-*` was missing until 3 Jul 2026, so every locale's preflight chunk (14 of them) was wrongly counted, inflating the measured size ~24 kB toward the ceiling. **When you add a namespace, add its `!â€¦/assets/<ns>-*.js` exclude.** The glob also excludes the dev-only `visual-diff` entry, `ResponsivePlayground` + `Playground*` dev chunks, and the six non-landing tab islands (`TranscriptPage`, `Toolbar`, `QuoteSections`, `QuoteThemes`, `CodebookPanel`, `AnalysisPage`) which now code-split into per-tab lazy chunks (only `Dashboard`, the landing route at `/report/`, loads on first paint and stays counted). Add new exclusions when introducing new dev-only or lazy-loaded chunks; if in doubt, leave it in the budget. Ratchet the limit down as optimisations land. The earlier 320 kB sum-of-everything metric overcounted by ~110 kB by including chunks that never load â€” see [docs/private/bundle-size-analysis-2026-05-02.md](../docs/private/bundle-size-analysis-2026-05-02.md) for the full breakdown
- **Island code-splitting â€” one dynamic owner per island.** The 6 tab islands (+ `Dashboard`) are `React.lazy`-loaded by their page wrappers (`src/pages/*Tab.tsx`); the `<Outlet/>` in `AppLayout` provides the shared `<Suspense fallback={null}>` boundary. **Do not also statically import an island from its page** â€” the legacy island mode in `main.tsx` already dynamically imports all of them, so a static import anywhere makes Vite/Rolldown emit `[INEFFECTIVE_DYNAMIC_IMPORT]` and silently fold the island back into the main/`components` chunk (it never code-splits, and the size gate creeps up). Each island needs exactly one owner that imports it dynamically. This was the root cause of the 215â†’220â†’230 kB ceiling creep fixed on 1 Jul 2026. A clean `npm run build` emits **zero** `INEFFECTIVE_DYNAMIC_IMPORT` warnings; if one reappears, a page has re-introduced a static island import
- **`npm run build` runs `tsc -b` which type-checks test files** â€” `tsconfig.json` includes `src/` which contains `*.test.tsx` files alongside source. Vitest has its own type context (looser), so tests may pass while `tsc -b` reports errors. Always run `npm run build` before committing frontend changes, not just `npm test`. Common culprits: `globalThis.fetch` (not `global.fetch`), window casts need `(window as unknown as Record<string, unknown>)` (double cast via `unknown`), and mock data must include all required type properties

## Tooling & dependency gotchas

- **Primary Node version is pinned in `.tool-versions` at the repo root** (currently `node 24` LTS â€” bump in one place, CI reads it via `setup-node`'s `node-version-file:`). ESLint 10 dropped Node 23; **Node â‰Ą 25 breaks jsdom's localStorage** (native Web Storage API conflict â€” confirmed on 25 and 26: Node's native `localStorage` global is a `--localstorage-file`-gated getter that resolves to `undefined` and shadows the Storage jsdom installs, so any test touching `localStorage` throws `Cannot read properties of undefined`). **If Vitest produces a large number of `localStorage` failures, the first thing to check is `node --version`** â€” not jsdom, not the test-setup (do NOT add a test-setup polyfill â€” the fix is the version pin). Run tests with `PATH="/opt/homebrew/opt/node@24/bin:$PATH"` if you default to a newer Node (mise/asdf aren't installed locally, so `.tool-versions` does not auto-apply to your shell). This cost a full hour of mis-diagnosis in Apr 2026 (140+ tests apparently failing on main; actually all passed once we switched to Node 24)
- **`--legacy-peer-deps` required for `npm install`** â€” `typescript-eslint@8.57.x` declares `typescript < 6.0.0` as peer dep, but works fine with TS 6. When they update their peer dep range, drop the flag
- **Vite 8 uses Rolldown** (Rust bundler) instead of esbuild+Rollup. Build is ~3x faster. `rollupOptions` in `vite.config.ts` is auto-converted by the compat layer. A clean build emits **no** `INEFFECTIVE_DYNAMIC_IMPORT` warnings â€” if one appears, a page wrapper has re-introduced a static island import that defeats code-splitting (see the island code-splitting bullet under Build & type-checking)
- **TypeScript 6 is stricter about mock types** â€” `ReturnType<typeof vi.fn>` no longer assignable to concrete function signatures. Use specific function types or `any` for test mocks
- **Vitest 4 `clearAllMocks` vs `restoreAllMocks`** â€” `restoreAllMocks` no longer reliably clears mock call counts in some scenarios. Use `clearAllMocks` in `beforeEach` when asserting call counts

## Architecture

- **`main.tsx` dual mode** â€” SPA mode (`#bn-app-root` exists â†’ `RouterProvider`) vs legacy island mode (dynamic `import()` â†’ individual `createRoot()` calls). The legacy path uses `Promise.all` for code splitting â€” islands load async, not synchronously like the old static imports. SPA mode is active in serve mode; legacy mode covers the static render path and transcript HTML files during transition
- **Module-level stores (not React Context)** â€” `QuotesContext.tsx`, `SidebarStore.ts`, and `ActivityStore.ts` use plain module-level state + `useSyncExternalStore` (React 18). Proven pattern, simpler than Context, doesn't depend on tree structure. `QuotesContext` owns quote mutations and toolbar filter state. `ActivityStore` owns background job tracking (e.g. AutoCode) â€” rendered from `AppLayout` so activity chips persist across tab navigation
- **Backward-compat shims** â€” `src/shims/navigation.ts` installs `window.switchToTab`, `window.scrollToAnchor`, `window.navigateToSession` as functions that delegate to React Router `navigate()`

## Gotchas

### React patterns

- **Clipboard copy after async fetch (Safari)** â€” `navigator.clipboard.writeText()` fails in Safari if called after an `await fetch()` because the user gesture context expires during the async gap. The fix: use `ClipboardItem` with a `Promise<Blob>` value and call `clipboard.write()` **synchronously** in the gesture handler. The browser resolves the promise internally while keeping the gesture alive. **Critical:** `clipboard.write()` must NOT be inside `.then()` â€” that makes it async and breaks Safari. Fallback for jsdom (no `ClipboardItem`): `writeText()` after fetch (acceptable in tests). See `ExportDropdown.tsx` `handleCopyQuotes`
- **Bulk actions on multi-selection** â€” `QuoteGroup.tsx` handlers (`handleToggleStar`, `handleToggleHide`, `handleTagAdd`) check `selectedIds` from FocusContext. If the clicked quote is in the selection, the action applies to all selected quotes. **Star direction**: click-based bulk uses the clicked quote's toggle intent (`newState`) as direction â€” click an unstarred star â†’ star all, click a starred star â†’ unstar all. This differs from the keyboard `s` shortcut in `useKeyboardShortcuts.ts`, which uses focused-quote heuristic (appropriate when there's no specific click target). **Hide**: bulk hides all selected with CSS collapse (no ghost animation), then clears selection. **Tag**: tag commit applies to all selected quotes with flash animation on each
- **Quote edit keydown handlers must `stopPropagation()`** â€” `handleCardKeyDown` (crop mode) and `handleEditKeyDown` (hybrid mode) in `QuoteCard.tsx` handle Enter/Escape. Without `stopPropagation()`, the keydown bubbles to the TimecodeLink `<a>`, the browser synthesizes a click, and `player.js`'s document-level click listener opens the video player. Both handlers need `preventDefault()` + `stopPropagation()`. See bug #9 in `docs/design-quote-editing.md`
- **`addTag()` dedup guard** â€” `QuotesContext.tsx` `addTag()` rejects case-insensitive duplicate tags (returns `prev` state unchanged). This is the single source of truth for all tag additions â€” fixes apply to manual adds, proposal accepts, etc.
- **Hidden-group tag UX** â€” when a codebook group is eye-toggled off, TagInput autocomplete shows a grey closed-eye icon (`EyeClosedIcon`, 12px SVG, `.tag-suggest-hidden-icon` CSS class) next to suggestions from hidden groups. On accept, `handleTagAdd` in `QuoteGroup.tsx` calls `toggleTagGroupHidden()` to auto-unhide the group so the badge is immediately visible. `allTagNames` prop (all tags including hidden-group ones) feeds the TagInput exclude list, preventing invisible re-adds. `hiddenTagNames` (derived from `hiddenTagGroups` + `tagGroupMap`) flows through QuoteGroup â†’ QuoteCard â†’ TagInput as `hiddenTags` prop
- **Clickable cards must be `<a>` not `<div>`** â€” any card that navigates on click (stat cards, nav links, etc.) must use an `<a>` element with a real `href` so that Cmd+click opens in a new tab. Use `onClick` with `if (e.metaKey || e.ctrlKey || e.shiftKey) return; e.preventDefault();` to keep programmatic React Router navigation for normal clicks while letting modifier-key clicks fall through to native browser behaviour. `statTargetToHref()` in `Dashboard.tsx` converts `"tab:anchor"` targets to pathname URLs. CSS needs `text-decoration: none; color: inherit` to prevent link styling
- **React Router navigation + scroll** â€” calling `window.switchToTab("quotes")` then `window.scrollToAnchor("section-x")` works via backward-compat shims (`src/shims/navigation.ts`). The scroll uses retry-aware polling (50 Ă— 100ms = 5s) because the destination page mounts and fetches data async. Anchor slugs must match between caller and target â€” AnalysisPage uses `toLowerCase().replace(/ /g, "-")` to match QuoteSections/QuoteThemes
- **Modal focus trapping via `useInert`** â€” `src/hooks/useInert.ts` sets `inert` on `#bn-app-root` while any portaled modal is open. Ref-counted (module-level counter) so overlapping modals don't remove `inert` prematurely. All 5 portal-based modals call `useInert(open)`: ModalNav (Help + Settings), ExportDialog, FeedbackModal, ThresholdReviewModal, AutoCodeReportModal. `resetInertCount()` exported for tests
- **Screen reader announcements** â€” `src/utils/announce.ts` provides `announce(message)` which sets text on a shared `aria-live="assertive"` region (`AnnounceRegion` component mounted in AppLayout). Used for star/hide/tag mutations (QuotesContext), tab navigation (AppLayout), and edit mode transitions (EditableText). Clears after 1s timeout. For tests: `resetAnnounce()` clears the module-level state
- **NavBar links are `<a>`, not `role="tab"`** â€” `NavLink` renders anchor elements. Tests use `getByRole("link", { name })` not `getByRole("tab", { name })`. The nav container is a plain `<nav>` (not `role="tablist"`). The native macOS toolbar uses tab semantics separately

### Routing & serve mode

- **Health payload shape (Mar 2026)** â€” `GET /api/health` includes footer config: `links.github_issues_url` and `feedback.{enabled,url}` in addition to `status` and `version`. Reuse the shared `HealthResponse` type (`src/utils/health.ts`) instead of ad-hoc inline interfaces to avoid shape drift across `Footer`, `FeedbackModal`, About panel, and export embedded data
- **Route priority in `app.py`** â€” transcript file route (`GET /report/sessions/{filename}` with `transcript_*.html` filter) must be defined before the SPA catch-all (`GET /report/{path:path}`). Starlette resolves routes in registration order. The `/report` â†’ `/report/` redirect is defined first, then transcript files, then the SPA catch-all, then the `StaticFiles` mount
- **`<!-- bn-app -->` markers** â€” wrap nav bar + all tab panels in `bristlenose/stages/s12_render/report.py`. The IIFE script block (including `BRISTLENOSE_ANALYSIS` JSON) is outside the markers and survives the serve-time substitution. `_REACT_APP_MOUNT` in `app.py` replaces these markers with `<div id="bn-app-root">`
- **IIFE-scoped globals need `window.` for React** â€” `bristlenose/stages/s12_render/report.py` wraps all JS in `(function() { ... })();`. Variables declared with `var` inside the IIFE are invisible to React (which runs in a separate ES module context). Any global that React needs must be explicitly assigned to `window.` inside the IIFE. Currently exposed: `window.BRISTLENOSE_VIDEO_MAP`, `window.BRISTLENOSE_PLAYER_URL`, `window.BRISTLENOSE_ANALYSIS`. If adding new globals for React, add `window.X = X;` after the `var` declaration â€” the `var` must stay for vanilla JS code in the same IIFE
- **Session route priority** â€” `/report/sessions/{filename}` handles both transcript HTML files (`transcript_*.html`) and React Router SPA routes (e.g. `/report/sessions/s1`). Non-transcript filenames serve the SPA HTML instead of raising 404, so React Router can handle client-side routing

### Video player

- **Two data paths** â€” video playback has two independent data flows: (1) **Static render path** â€” `html_helpers._build_video_map()` reads `InputSession.files` (full absolute paths) and writes `window.BRISTLENOSE_VIDEO_MAP` with correct `file://` URIs into the IIFE. `player.js` reads this via event delegation. (2) **Serve/SPA path** â€” the importer reads transcript headers (`# Source: interviews/show and tell 40.mov`) which store **relative paths including subdirectory** (written by `merge_transcript.py` using `fpath.relative_to(input_dir)`), stores them in the DB, and the video-map API (`/api/projects/{id}/video-map`) converts DB paths to `/media/` URIs via `_file_to_media_uri()`. `PlayerContext.tsx` fetches this API. **Encoding rule**: `_file_to_media_uri()` returns **unencoded** paths (e.g. `/media/interviews/show and tell 40.mov`). JavaScript's `encodeURIComponent()` in `PlayerContext.tsx` handles encoding once when building the player URL hash. Never add `urllib.parse.quote()` in Python â€” that causes double-encoding (`%20` â†’ `%2520`). In SPA mode, `player.js` bails out (`if (document.getElementById('bn-app-root')) return`) â€” only `PlayerContext` handles playback
- **Glow index keying: session ID vs speaker code** â€” transcript segments have `data-participant` set to the speaker code (`m1`, `p1`), but the popout player sends `pid=sessionId` (`s1`) in `bristlenose-timeupdate` messages (because `TimecodeLink` passes `data.session_id` to `seekTo`). `buildGlowIndex` in `PlayerContext.tsx` extracts the session ID from the URL pathname (`/report/sessions/:sessionId`) and keys all transcript segment entries under that ID. Without this, the glow index would be keyed by speaker code and no entries would match the player's session-ID-based lookup
- **Progress bar on transcript segments** â€” `PlayerContext.tsx` sets a `--bn-segment-progress` CSS custom property (0â€“1) on the active `.transcript-segment` during playback. The CSS `::before` pseudo-element in `timecode.css` renders a 3px left border that fills top-to-bottom via `scaleY()`. The segment needs `position: relative` and `overflow: hidden` for this to work. `prefers-reduced-motion` disables the transition

Preserved across `put_tags` bulk-replace (snapshot before delete, restore on re-insert in `routes/data.py`). Exposed in quotes API as `source` on `TagResponse`. ### Testing

- **Module-level stores persist across tests** â€” always call `resetStore()` / `resetSidebarStore()` / `resetPlaygroundStore()` / `resetActivityStore()` in `beforeEach`
- **`vi.mock` factory functions can't reference variables declared after them** (mocks are hoisted) â€” inline data directly
- **`useRef<T>()` needs explicit `undefined` argument** in newer TypeScript: `useRef<T>(undefined)`
- **Mock `getCodebook`** with `.mockResolvedValue()` in `beforeEach`, not just in `vi.mock` factory (`restoreAllMocks` resets it)
- **Router tests**: use `createMemoryRouter` with `routes` array (exported from `router.tsx`). Use `getByRole("link", { name })` not `getByText` to find NavBar links (they're `<a>` elements, not tabs)
- **Playwright E2E tests** (`e2e/`) â€” 3 test layers (console errors, link crawler, network assertions) running against the smoke-test fixture on Chromium + WebKit. `playwright.config.ts` auto-starts `bristlenose serve` via `webServer` â€” locally uses `.venv/bin/bristlenose` (resolved path), in CI uses bare `bristlenose` (on PATH). Port 8150, overridable via `BN_E2E_PORT`. The link crawler skips `<a role="button">` elements. Known bugs are `test.fixme()` stubs. See `docs/design-playwright-testing.md`
- **`vi.useFakeTimers()` + `requestAnimationFrame` mocking** â€” fake timers replace `globalThis.requestAnimationFrame` / `cancelAnimationFrame`. If a hook uses `window.requestAnimationFrame` (not bare `requestAnimationFrame`), mocks must be installed on `window` too. Use the `installRafMock()` helper pattern: call it AFTER `vi.useFakeTimers()` to re-install synchronous rAF/cAF on both `globalThis` and `window`. React's passive effect cleanup runs after test teardown, so `window.cancelAnimationFrame` may be gone by then â€” the hook must guard: `if (typeof window.cancelAnimationFrame === "function")`. See `useScrollSpy.test.ts` for the canonical example
- **Safari `getBoundingClientRect()` during smooth scroll** â€” Safari returns physically impossible values (bottom < top, negative height) from `getBoundingClientRect()` while `scrollIntoView({ behavior: "smooth" })` animation is in progress. Don't trust `getBoundingClientRect()` for at least 600ms after triggering a smooth scroll. The `useScrollSpy` two-phase override is the canonical workaround

### Sidebar & playground

- **Sidebar layout** â€” 6-column CSS grid (`SidebarLayout.tsx`): `[toc-rail | toc-sidebar | center | minimap | tag-sidebar | tag-rail]`. - **Desktop embedded: rails removed** â€” when `isEmbedded()`, `SidebarLayout` adds the `embedded` class, gates out both close-Ă— buttons, and redirects focus-on-close to `.center` (the hidden rail button can't take focus; `.center` gets `tabIndex={-1}` in embedded only). - **Rail drag-to-open** â€” dragging from the closed rail reveals the sidebar as a fixed overlay that tracks the pointer 1:1. - **Overlay auto-close toggle** â€” `TocSidebar` reads `pg.overlayAutoClose` from PlaygroundStore. Default `false` (overlay stays open after clicking a TOC link â€” user closes by mousing out). When `true`, the panel closes after 400ms (legacy behaviour). Configurable via the dev playground
